## Enforcement-Mechanism Findings (basis for the plan)

The PowerShell coverage gate is enforced at two layers; both must honor the exclusion for the re-audit to reach `blocking_count == 0`:

1. **Machine gate — SubagentStop hook.** `.claude/hooks/validate-feature-review-coverage.ps1` function `Get-ChangedLanguageSet` (lines 121-138) parses `artifacts/pr_context.summary.txt`, matching lines of the form `- <path> (+N/-N)`, and maps `.ps1`/`.psm1` extensions to the `PowerShell` changed-language set. Because the 15 changed paths under the "Changed files overview" section are all `.claude/hooks/*.ps1`, PowerShell is enumerated as a changed language and a coverage verdict is required. The hook has no config-driven exclusion list today, so the exclusion must be applied inside `Get-ChangedLanguageSet` (skip paths matching `.claude/hooks/**`). This is the only authoritative location where the machine gate stops treating the hooks as in-scope application code.
2. **Policy / agent-judgment layer.** `.claude/skills/feature-review-workflow/SKILL.md` (coverage-scope section, step 5 and the Constraints/Scope-Invariant clauses) and `.claude/rules/general-unit-test.md` / `.claude/rules/quality-tiers.md` govern the human-readable PASS/FAIL verdict. The exclusion rationale must be stated in the canonical policy files so the verdict is authored as repository policy, not asserted at review time.

**Self-reference hazard (handled).** The file that must be edited at the machine-gate layer, `validate-feature-review-coverage.ps1`, is itself a `.claude/hooks/*.ps1`. The exclusion is declared for the whole `.claude/hooks/**` glob, which includes the edited file; therefore editing it does not re-trigger the PowerShell-in-scope enumeration, because every `.claude/hooks/**` path is filtered out of the changed-language set regardless of presence in the diff.

**`.claude/rules/` policy-edit note.** The `policy-compliance-order` baseline states "Do NOT modify policy documents under `.claude/rules/`." The operator decision (Option B) explicitly directs authoring the coverage-scope exclusion as repository policy in `.claude/rules/general-unit-test.md` and/or `.claude/rules/quality-tiers.md`. The remediation inputs prohibit only lowering or removing the product-code coverage thresholds; adding a scope-exclusion clause for harness tooling does not lower product-code thresholds. This plan treats the operator's Option B as the authoritative override for these two files and records the override explicitly in P1-T2/P1-T3 evidence. No threshold value is changed.
