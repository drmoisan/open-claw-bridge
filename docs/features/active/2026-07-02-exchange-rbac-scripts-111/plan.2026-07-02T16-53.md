## Conventions Used by This Plan

- `FEATURE` = `docs/features/active/2026-07-02-exchange-rbac-scripts-111`
- `MODULE` = `scripts/powershell/modules/OpenClawRbac`
- `TESTS` = `tests/scripts/powershell/modules/OpenClawRbac`
- `<ts>` = ISO-8601 timestamp `yyyy-MM-ddTHH-mm` captured at artifact-creation time.
- All evidence artifacts live under `FEATURE/evidence/<kind>/` (canonical kinds: `baseline/`, `qa-gates/`, `regression-testing/`, `issue-updates/`, `other/`). No evidence may be written under `artifacts/` except that raw Pester/coverage output files emitted by the PoshQC toolchain may land in `artifacts/pester/` as tool output; the evidence summary artifact under `FEATURE/evidence/<kind>/` is authoritative and must record the numeric values.
- Every command-step evidence artifact must contain: `Timestamp:`, `Command:`, `EXIT_CODE:`, `Output Summary:` (1-20 lines with the essential result signal). Test-step artifacts must include numeric coverage headline values in `Output Summary:`.
- **PoshQC batch toolchain loop** (referenced by Phases 1-3): run `mcp__drm-copilot__run_poshqc_format` → `mcp__drm-copilot__run_poshqc_analyze` → `mcp__drm-copilot__run_poshqc_test` in that order, scoping scan folders to `MODULE`, `scripts/Invoke-OpenClawExchangeRbacSetup.ps1`, `TESTS`, and `tests/scripts/Invoke-OpenClawExchangeRbacSetup.Tests.ps1` where the tool supports folder scoping (fall back to repo default scope where it does not). If any step fails or changes files, restart the loop from format. A batch is complete only when all three steps pass in a single uninterrupted pass. `EXIT_CODE: SKIPPED` is never a passing outcome for any command task in this plan.
- Mocking: tests mock only the nine `Invoke-OpenClaw*` wrapper functions with `Mock -ModuleName OpenClawRbac`; never mock Exchange cmdlets directly; mock signatures must match production wrapper named parameters exactly (mock signature parity per `.claude/rules/powershell.md`). No temp files in tests. Every file under the 500-line cap.
