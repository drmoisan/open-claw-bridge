## Acceptance Criteria Evaluation

### AC-1 — Webhook endpoint: **PASS**

- Unchanged by the remediation cycle and byte-identical across the rebase; the 02-34 evaluation evidence holds and all supporting tests pass in the reviewer's fresh 716/716 Core.Tests run at `df0151a` (2026-07-06).
- Handshake: `NotificationRequestProcessor.HandleHandshake` returns 200/`text/plain` with `WebUtility.HtmlEncode(WebUtility.UrlDecode(token))` — unit test plus factory round-trip through the real route.
- 202-and-drop: unknown subscription, mismatched clientState (constant-time compare), and missing `resourceData.id` each drop with Warning while the batch returns 202; malformed JSON returns 400 in the repo `{code, message}` shape.
- Exactly-one enqueue asserted with the `{Mailbox, MessageId, ChangeType}` item shape; lifecycle DataRows cover all three events. The processor has no `HttpClient`/executor dependency (no Graph I/O by construction).

### AC-2 — Subscription manager: **PASS**

- Request shapes pinned over mocked `HttpMessageHandler` (POST `subscriptions` with changeType/resource/URLs/clientState/ISO-8601 expiration; PATCH with `expirationDateTime` only).
- Deterministic renewal-due via pure `IsRenewalDue(now, expiration, lead)` with the boundary asserted exactly at `expiration - lead`, plus 3 CsCheck properties; lifetime cap 10,080 and lead rules enforced by `CloudSyncOptionsValidator` with boundary tests.
- Lifecycle routing tests for all three routes plus the unknown-event guard and `reauthorize_failed` marking.
- Persistence/restart: `graph_subscriptions` round-trip, status update, delete idempotency, double schema-ensure, restart survival via a fresh repository instance. Strengthened by the cycle: the corrupt-expiration read path now fails fast (`InvalidOperationException` naming id and column) instead of coercing to `MinValue` — reviewer code-read at head plus the directed corrupt-row test.

### AC-3 — Delta reconciler: **PASS**

- Multi-page walk pins the initial URL (including `$select` reuse of `MessageSelect`), verbatim nextLink follows, terminal deltaLink persistence, resume-from-stored-link, `MaxPages` bound with failed ingest run.
- Sink identity: upserts flow through `CoreCacheRepository.UpsertMessagesAsync` with the D-3 ready/graph status, asserted by row inspection; `@removed` entries skipped (now including the id-less variants added by the cycle).
- Re-sync triggers: empty stored link starts from the initial request; `TriggerResyncAsync` re-syncs even when a link is stored.
- Defensive parse arms hardened by the cycle: page-without-`value`, id-less `@removed`, unparseable entry -> `TRANSPORT_FAILURE` + failed `delta_reconcile` run. Residual: the literal-null entry arm is structurally unreachable dead code (CR-117-05, Major, follow-up routed — a code-quality latent defect on a degenerate off-contract input, not a failure of this criterion's specified behaviors, which are all demonstrated).
