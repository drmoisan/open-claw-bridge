## Architecture decision (authoritative for this plan)

The deterministic agent components are folded into the EXISTING `OpenClaw.Core` project under `src/OpenClaw.Core/Agent/**`, in the `OpenClaw.Core.Agent` namespace (a namespace, not a project). No new `.csproj` is created and no new `ProjectReference` edge is added anywhere. This is required by `.claude/rules/architecture-boundaries.md` rule 6, which states `OpenClaw.Core` may depend only on `OpenClaw.HostAdapter.Contracts`; a new `OpenClaw.Core -> OpenClaw.Core.Agent` project reference would violate that uniform, PR-blocking gate, and policy files under `.claude/rules/` must not be modified. A separate-project approach (the earlier OR-1 reading) is therefore rejected.

Namespace partition (precise, for the AC-10 architecture assertion):

- Deterministic pure-logic surface (D1–D4): types under `src/OpenClaw.Core/Agent/**` in the `OpenClaw.Core.Agent` namespace, EXCLUDING the `OpenClaw.Core.Agent.Runtime` namespace. These are the normalizer, triage engine, dependency scorer, priority/recurrence/move classifiers, and slot proposer. They must NOT depend on `OpenClaw.MailBridge*`, `OpenClaw.HostAdapter*`, `Microsoft.Office.Interop.Outlook*`, or `System.Runtime.InteropServices` COM-interop helpers.
- Contracts surface (D5/D6 options, interface, DTOs): types under `src/OpenClaw.Core/Agent/Contracts/**` in the `OpenClaw.Core.Agent` namespace. Same no-MailBridge/no-HostAdapter/no-COM constraint as the deterministic surface.
- Runtime seam (exempt): types under `src/OpenClaw.Core/Agent/Runtime/**` in the `OpenClaw.Core.Agent.Runtime` namespace — the HostAdapter-backed `ISchedulingService` adapter, the OR-4 DTO mapper, and the `SchedulingWorker`. These are explicitly exempt from the boundary assertion; they are the seam implementation and may reference `OpenClaw.HostAdapter.Contracts` (which `OpenClaw.Core` already references), fully rule-6-compliant.

The contract-parity invariant (AC-10 / AC-U1) is enforced by a namespace-scoped `NetArchTest.Rules` assertion in `tests/OpenClaw.Core.Tests` (the existing project that already references `OpenClaw.Core`), not by assembly isolation. Rule 6 and COM-confinement rule 3 are preserved because no project reference is added.

## Toolchain commands (per `.claude/rules/csharp.md`)

- Format: `dotnet csharpier .` (CSharpier; do not use `dotnet format`).
- Lint + type-check: `dotnet build OpenClaw.MailBridge.sln -c Debug -p:EnableNETAnalyzers=true -p:EnforceCodeStyleInBuild=true -p:TreatWarningsAsErrors=true`.
- Architecture: enforced by the existing project graph (no new `ProjectReference` is added) plus the namespace-scoped NetArchTest assertion added in Phase 1 (P1-T7) in `tests/OpenClaw.Core.Tests`.
- Test + coverage: `dotnet test OpenClaw.MailBridge.sln -c Debug --settings mailbridge.runsettings --collect:"XPlat Code Coverage"`.

Each phase ends with the full ordered toolchain loop (format → lint → type-check → architecture → unit/property tests) and must be green before the next phase begins. Restart the loop from formatting whenever any stage fails or rewrites a file.

## Coverage policy (mandatory, per `.claude/rules/general-unit-test.md` + `quality-tiers.md`)

- Uniform gates across tiers: line coverage >= 85%, branch coverage >= 75% on agent application code (the folded `OpenClaw.Core.Agent` namespace within `OpenClaw.Core`).
- T1 agent code requires >= 1 property-based test per pure function (`Normalize`, `DependencyScorer.Score`, `TriageEngine.Triage`, `OwnerPriorityClassifier.Classify`, `RecurringMeetingClassifier.Classify`, `MovePolicy.CanMove`, `SlotProposer.ProposeTimes`).
- Time-dependent tests use `FakeTimeProvider` (`Microsoft.Extensions.TimeProvider.Testing`); no `Thread.Sleep`/`Task.Delay`/temp files in tests.
- Baseline and final-QC coverage values must be recorded as numeric headline values, not placeholders.
