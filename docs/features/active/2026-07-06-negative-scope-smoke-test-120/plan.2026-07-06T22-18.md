## Conventions Used by This Plan

- `FEATURE` = `docs/features/active/2026-07-06-negative-scope-smoke-test-120`. All evidence artifacts are written under `FEATURE/evidence/<kind>/` per `.claude/skills/evidence-and-timestamp-conventions/SKILL.md`. No `artifacts/` evidence path is permitted.
- Every command-step evidence artifact MUST contain `Timestamp:` (ISO-8601 `yyyy-MM-ddTHH-mm`), `Command:`, `EXIT_CODE:`, and `Output Summary:`. Test-step artifacts additionally record numeric line and branch coverage headlines in `Output Summary:`.
- CSharpier invocation form: the global tool `csharpier check .` / `csharpier format .` (the repository has no working local `dotnet csharpier` tool manifest; do not use the `dotnet csharpier` driver and do not use `dotnet format`).
- `dotnet build` provides the type-check stage via nullable reference-type analysis (`<Nullable>enable</Nullable>` in each csproj). This repository has no `Directory.Build.props`, no `.editorconfig`, and no third-party analyzer package stack, and `TreatWarningsAsErrors` is not set; therefore `dotnet build` does not run a third-party lint/analyzer gate and exits 0 even when nullable warnings are present. The executor treats any nullable (or other compiler) warning in the build output as a must-fix defect by inspecting the build log, since the exit code will not fail on warnings.
- Architecture-boundary tests run inside the MSTest suite (existing `*ArchitectureBoundaryTests` pattern) and are exercised explicitly via `dotnet test --filter "FullyQualifiedName~ArchitectureBoundary"`.
- Observed test harness (verified by reading `tests/OpenClaw.Core.Tests/`): MSTest + Moq + FluentAssertions + CsCheck + `FakeTimeProvider`, with the shared `FakeHttpHandler` fixture defined in `tests/OpenClaw.Core.Tests/HostAdapterHttpClientTests.cs`. New tests follow this observed harness (known, accepted divergence from the xUnit/NSubstitute wording in `.claude/rules/csharp.md`, per spec Constraints & Risks).
- Toolchain loop rule (applies to every phase-closing loop task and to Phase 6): format -> build (compile + nullable type-check; no third-party analyzer gate present) -> architecture tests -> unit/property/contract tests. Restart from format if any stage fails or changes any file; the loop is complete only when all stages pass in a single pass. Because warnings do not fail the build, the loop is complete only when the build log shows zero warnings in addition to a zero exit code.
