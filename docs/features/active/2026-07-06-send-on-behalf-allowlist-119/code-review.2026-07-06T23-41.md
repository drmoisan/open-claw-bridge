## Executive Summary

This is a small, well-scoped T1 security change: a fail-closed principal-mailbox allowlist gating Graph send-on-behalf, delivered as one new pure function, one additive options property, one shape-only validator rule, and one gate in `SendMailAsync`. The implementation matches the spec's seven-row decision table exactly, and the three security-critical properties the caller asked to verify all hold under direct code inspection and reviewer-run tests:

1. **Zero-I/O deny.** The gate runs before `executor.ExecuteAsync`; on `DeniedNotAllowlisted` the method returns `Task.FromResult` with the deny envelope (`GraphHostAdapterClient.SendMail.cs:43-66`). Since `GraphRequestExecutor.ExecuteAsync` is what acquires the token, neither token acquisition nor HTTP can occur. The tests prove it decisively: the deny tests use a `MockBehavior.Strict` `IAppTokenProvider` with **no setups** (any call throws) and a counting handler asserted at zero invocations. Fail-before evidence shows exactly these three tests failing (EXIT 1) before the gate existed and passing (EXIT 0) after.
2. **Single decision source.** `ComposeSendMailBody` takes the `SendAuthorizationDecision` as a parameter and injects `from` iff `AllowedOnBehalf` (`SendMail.cs:90-105`); the baseline's inline `principalIsAssistant` re-derivation was removed. The predicate cannot diverge from the authorization decision.
3. **Fail-closed default.** `AllowedPrincipalMailboxUpns { get; } = new List<string>()` — an empty collection — and the authorizer returns `DeniedNotAllowlisted` for every `{p} != {a}` against an empty list. The CsCheck deny-completeness property (`iter: 1000`) generalizes this beyond the directed cases. The error surface is exactly as specified: `UNAUTHORIZED` / `SendOnBehalfDenied` / `Retryable=false`, message naming the config key with no UPN echo, one warning log carrying the request id only.

The NetArchTest CloudGraph boundary rules pass (reviewer isolation run 3/3); the new type is `internal` and falls under the existing namespace rules automatically. Toolchain is clean in a single pass (CSharpier 325 files EXIT 0; build 0/0; 1192/1192 solution tests). Coverage: new file 100.00% line / 87.50% branch; both modified logic files 100.00%/100.00%; no async-instrumentation masking on this branch (the gate is synchronous and fully instrumented).

**Findings:** 0 Blocking, 0 Major, 2 Minor, 3 Info. No remediation required.

## Code Quality Observations (non-findings)

- **Test design quality on the deny path is the strength of this branch.** The `DenyClient` helper's strict no-setup token provider makes any token acquisition a hard failure rather than a missed verification — a stronger proof than `Times.Never` alone (which the tests also assert). The `NeverInvokedHandler` counts rather than throws, giving a readable zero-invocation assertion with a because-message.
- **Log hygiene is enforced, not just claimed:** the `RecordingLogger` assertion checks both the positive (exactly one warning containing the request id) and the negative (neither UPN appears in the formatted message), and the deny-envelope message assertion does the same. This matches the executor-precedent no-UPN-in-logs convention.
- **Helper-seeded allowlists in existing test files are the correct regression pattern:** existing on-behalf tests keep exercising the send path under the new gate because the shared `Client`/`Service` helpers allowlist the configured principal, while every existing test method body is textually unmodified (reviewer-verified against the diff hunks). No assertions were weakened.
- **The validator rule preserves the full-violation-list style** (`violations.Add`, no early return) and the message convention (names the key, echoes no configured value), consistent with the surrounding rules in the same file.
- **`Recased`/`Padded`/`RecasedList` generator combinators** in the property tests are minimal and deterministic under CsCheck's seeded sampling; the accumulator-based `RecasedList` avoids shared mutable state across generated samples by copying (`Append` returns a fresh list).
