## Overview

This plan delivers the large-path, additive OpenClaw pre-MVP deployment model described by the feature folder. It preserves the existing Windows-local named-pipe bridge, reuses `OpenClaw.MailBridge.Client` as the first transport seam, and adds the HostAdapter, Core, Docker/devcontainer assets, documentation, tests, and coverage evidence required for full-feature execution.

## Assumptions

- `OpenClaw.HostAdapter` and `OpenClaw.Core` are implemented on `net10.0`; the existing Windows bridge and CLI remain on `net10.0-windows`.
- This plan does not modify PowerShell scripts or PowerShell tests. If executor-side implementation expands into `scripts/*.ps1` or PowerShell test files, the plan must be amended before Phase 0 to add PowerShell baseline and final-QC tasks.
- This plan does not modify `.github/workflows/*.yml` files. If workflow changes become necessary, add explicit `actionlint` baseline and final-QC tasks before execution continues.
- Coverage delta commands use `git merge-base HEAD main`; executor-side preflight must confirm `main` is still the correct comparison base before those tasks run.
